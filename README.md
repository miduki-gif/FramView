# オリジナルプロダクトテーマ
 - 要件の決まり具合を見えるかして後出し変更を減らすアプリ
## 課題
 - アプリやシステム開発を行う際に後から追加や変更が発生したり、
どこからが追加や変更の範囲かが曖昧になりトラブルや工数が発生してしまうこと
## アプリによって解決できること
 - どこから追加や変更の範囲なのかが最初の段階で視覚的に共通認識を持つことができる
 - どこまで決まっていて、決まっていないのはどこなのか等要件整理をあらかじめ行うことができる
## キャッチコピー
 - 要件の決定度を可視化
## 具体的には
 - どこまで決まっていて、どこからが未決定か項目ごとに決定度を示すことにより視覚的に可視化する
 - 質問に答えるだけでどんな要件が必要か形になる

## できること
 - どんなアプリやシステムが作りたいのか質問事項に対して選択形式で選択し(オプション等はyes,no)、
 やりたいことを引き出す
 - 選択してもらった回答で必要なものの機能が一覧で表示
 - 機能一覧で表示した機能をプレビュー画面で簡易的に確認できる
 - 各項目の決定状況と要件決定度をステータスで表示できる
## どんな機能が必要
 - どんなことをしたいのかをヒアリングするために以下のパターンに分けて質問をする
  - 既存業務を改善・アップデートしたい
   - 新規事業や新しい取り組みをしたい
  - 既存業務の場合
   - 新しい取り組みや新規事業の場合
